# 換新電腦設定指南

## 需要複製的資料夾
把整個 `D:\claude_司` 複製到新電腦的同樣路徑。
**注意：`設計需求系統\node_modules` 不需要複製**（太大，到新電腦重新安裝即可）。

---

## 新電腦安裝步驟

### 1. 安裝 Node.js
前往 https://nodejs.org 下載並安裝 LTS 版本（目前 v20+）

### 2. 安裝專案套件
開啟終端機，進入專案資料夾：
```
cd D:\claude_司\設計需求系統
npm install
```

### 3. 確認 .env.local 已複製
`.env.local` 檔案在 `設計需求系統\.env.local`，裡面有：
- Supabase URL 和 API Key（資料庫連線）
- Google Service Account（試算表同步）
- 其他設定

### 4. 啟動開發伺服器
```
npm run dev
```
然後開瀏覽器到 http://localhost:3030

---

## 常用指令

| 指令 | 用途 |
|------|------|
| `npm run dev` | 啟動開發伺服器（port 3030） |
| `npm run seed:remittances` | 重新匯入匯款歷史資料到 Supabase |
| `npm run seed:cooperations` | 重新匯入合作紀錄到 Supabase |
| `npm run seed:design-requests` | 重新匯入設計需求到 Supabase |
| `npm run sync:sheets` | 同步設計需求到 Google Sheets |
| `node scripts/sync-remittances-to-sheets.mjs` | 同步匯款資料到 Google Sheets |

## 如果資料庫需要重建（極少發生）

在 Supabase Studio → SQL Editor 依序執行：
1. `supabase/migrations/001_initial.sql`
2. `supabase/migrations/002_tighten_rls.sql`
3. `supabase/migrations/003_design_request_import.sql`
4. `supabase/migrations/004_cooperations.sql`
5. `supabase/migrations/005_add_designers.sql`
6. `supabase/migrations/006_remittances.sql`

然後執行：
```
npm run seed:cooperations
npm run seed:remittances
```
